    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><h1><a id="user-content-gcd-project" class="anchor" href="#gcd-project" aria-hidden="true"><span class="octicon octicon-link"></span></a>GCD-project</h1>

<p>Project for Getting and Cleaning Data, Coursera.</p>

<p>The purpose of this project was to demonstrate our ability to collect, work with, and clean a data set. We had to prepare tidy data that can be used for later analysis. The project has to obtain a non-tidy dataset of accelerometers data from the Samsung Galaxy S smartphone (see: <a href="http://archive.ics.uci.edu/ml/datasets/Human+Activity+Recognition+Using+Smartphones">http://archive.ics.uci.edu/ml/datasets/Human+Activity+Recognition+Using+Smartphones</a>).</p>

<p>The course instructions were the following:</p>

<ol>
<li>You should create one R script called run_analysis.R that does the following. </li>
<li>Merges the training and the test sets to create one data set.</li>
<li>Extracts only the measurements on the mean and standard deviation for each measurement. </li>
<li>Uses descriptive activity names to name the activities in the data set</li>
<li>Appropriately labels the data set with descriptive activity names. </li>
<li>Creates a second, independent tidy data set with the average of each variable for each activity and each subject. </li>
</ol>

<p>This Github repository provides a script (<em>run_analysis.R</em>), a tidy dataset (<em>tidymeans.txt</em>), and a codebook (<em>CodeBook.md</em>). </p>

<h2><a id="user-content-run_analysisr" class="anchor" href="#run_analysisr" aria-hidden="true"><span class="octicon octicon-link"></span></a>run_analysis.R</h2>

<p>The script was implemented under Microsoft Windows 8.1 and RStudio. It was not tested under any *nix environment nor the command-line version of R. Please run it from RStudio. Be aware that the script takes about 5 minutes to complete all the steps, including downloading the original dataset, on a Core i7 laptop and a ~15Mbit/s ADSL.</p>

<p>The script performs the following high-level tasks:</p>

<ul>
<li>Prepares the R dependencies and the working directories</li>
<li>Downloads and unzip the original dataset (if not already present)</li>
<li>Loads the features, keeps only the variable related to mean and std</li>
<li>Loads the subjects and activities. It merges them.</li>
<li>It merges features, subjects, and activities</li>
<li>Cleans the variable names to human readable form (and R coding standard compliant)</li>
<li>Creates the tiny data set with the average of each variable for each activity and each subject. </li>
</ul>

<p>Please note that the steps are documented at a deeper level in CodeBook.md.</p>

<h2><a id="user-content-codebookmd" class="anchor" href="#codebookmd" aria-hidden="true"><span class="octicon octicon-link"></span></a>CodeBook.md</h2>

<p>The CodeBook contains a description of the original dataset, the data acquisition process, the data transformation process, the variables, and the tidy dataset format.</p>
